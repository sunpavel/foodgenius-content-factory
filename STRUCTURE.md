## Модели и источники данных
- **Текст / концепт:** `gemini-2.5-flash`
- **Тренды:** YouTube Data API (топ по просмотрам) + Gemini Google Search grounding (факты + источники)
- **Видео:** `veo-3.0-fast-generate-001` (8-сек клипы, вертикаль)
- **Картинки (Nano Banana):** `gemini-2.5-flash-image` — ЕЩЁ НЕ ИСПОЛЬЗУЕТСЯ
- **Озвучка (TTS):** ещё не подключено
- **Музыка:** нет

## Текущий поток данных (и где РАЗРЫВ)
```
WF#1 концепт (JSON со сценами)  ✅
      │  ← РАЗРЫВА НЕТ оркестрации: концепт не передаётся дальше автоматически
      ▼
WF#2 Veo: генерит ТОЛЬКО 1 клип по хардкод-промпту, не по сценам концепта  ⚠️
      │  ← нет: цикла по всем сценам, консистентного героя, image-to-video
      ▼
Сборка ffmpeg: умеет выжечь 1 хук+CTA на 1 клип  ⚠️
      │  ← нет: склейки нескольких сцен, таймингов субтитров, пекшота, музыки, озвучки
      ▼
Telegram доставка  ✅
```

## Что РАБОТАЕТ
- Концепты на реальных трендах (хорошего качества).
- Veo генерит видео на сервере (билинг ок).
- ffmpeg выжигает русский текст; путь n8n→сервер→Telegram безопасен (секреты только в n8n).

## Чего НЕТ (честно)
- Нет сквозного конвейера: концепт → все клипы → сборка → ролик.
- Veo-клипы пока слабые (абстрактные), без консистентного героя.
- Нет музыки, нет озвучки, нет мультисцена-сборки с субтитрами и пекшотом.
- Нет автозалива в YouTube/соцсети.
- Реального готового ролика-рекламы пока не существует.
